온라인 배포 할 때
config - settings.py에서 DEBUG = False

DEBUG = False (운영서버에서 작동)  - 웹서버(리눅스)에서 관리
-> static(정적파일) 관련 - 이미지, js, css
template 관련 파일 관리 - WAS(웹 애플리케이션 서버 - 장고, 플라스크, 스프링)

DEBUG = True (개발서버에서 작동)


1. get_object_or_404()의 기능 : url 경로 틀리면 페이지 없음(404코드 출력)처리가 되는 기능
2. csrf_token
3. 성능 테스트 항목
- 평균 cpu 점유율
- 평균 응답 시간
- 동시 사용자 수
- TPS
4. 단위 테스트
- 작은 소프트웨어 단위(컴포넌트 또는 모듈)를 테스트, 개발자 자신에 의해 행해짐
5. 단위 테스트 방법
- 구조 기반 : 프로그램 내부 구조 및 복잡도를 검증 - 화이트 박스 테스트
- 명세 기반 : 목적 및 실행 코드 기반의 실행을 통한 - 블랙 박스 테스트
